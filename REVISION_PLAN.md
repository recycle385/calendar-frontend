# [기획팀] 상세 실행 계획: 검토 후 개선 사항

## 1. 개요
CEO의 요청에 따라 사용자 경험(UX) 개선 및 인증 안정성 확보를 위한 3가지 주요 기능 보완 작업을 수행합니다.

## 2. 상세 요구사항 명세

### ① 홈페이지 로그인 버튼 고도화 (Conditional Header)
- **목적:** 로그인 여부에 따른 동적 UI 제공으로 사용자 편의성 증대.
- **요구사항:**
    - `AuthContext`를 통해 전역 인증 상태를 구독.
    - 비로그인 시: "로그인" 버튼 표시 (`/login` 연결).
    - 로그인 시: "내 대시보드" 또는 "마이페이지" 버튼으로 변경 (`/dashboard` 연결).
- **디자인 가이드:** Soft Kawaii 스타일 유지 (둥근 모서리, 파스텔 컬러).

### ② 인증 리다이렉트 루프 해결 (Auth Stability)
- **목적:** 로그인 후 `/login` 페이지로 튕기거나 머무르는 현상 제거.
- **요구사항:**
    - `AuthContext` 내에 `user`, `accessToken`, `loading` 상태 관리.
    - `PublicRoute`(`/login`) 진입 시 `accessToken`이 존재하면 즉시 `/dashboard`로 리다이렉트.
    - `AuthCallbackPage`에서 토큰 저장 후 컨텍스트 상태를 즉시 동기화.
    - `api/client.ts`의 401 에러 핸들러와 `AuthContext` 초기화 로직 연동.

### ③ 일정 투표 캘린더 UI 전환 (Calendar Voting UI)
- **목적:** 리스트 형태의 날짜 선택을 캘린더 뷰로 변경하여 가독성 및 직관성 향상.
- **요구사항:**
    - `CalendarVotePage`의 날짜 선택 영역을 7열 그리드 월간 캘린더로 변경.
    - **활성화 조건:** 백엔드에서 내려온 `dates`에 포함된 날짜만 선택 가능하도록 활성화. 그 외 날짜는 비활성화(Dimmed) 처리.
    - **인터랙션:** 날짜 클릭 시 다중 선택 지원. '가능해요'(Secondary), '미정이에요'(Accent) 상태에 따른 색상 변화 적용.
    - **모바일 대응:** 모바일 환경에서 터치 영역(Hit area)을 충분히 확보한 큼직한 날짜 셀 디자인.

## 3. 개발/디자인 실행 로드맵

1. **Phase 1: 인프라 보완 (개발)**
    - `src/contexts/AuthContext.tsx` 구현 및 `App.tsx` 래핑.
    - 전역 상태 기반 인증 가드(`AuthRoute.tsx`) 리팩토링.
2. **Phase 2: UI/UX 개선 (개발/디자인)**
    - `LandingPage.tsx` 헤더 조건부 렌더링 적용.
    - `CalendarVotePage.tsx` 내 캘린더 그리드 UI 컴포넌트 개발.
3. **Phase 3: 검증 및 배포**
    - 인증 시나리오별 리다이렉트 정상 작동 확인.
    - 캘린더 UI 투표 기능 정상 작동 확인.

## 4. 기대 효과
- 사용자 인증 상태의 일관성 유지로 이탈률 감소.
- 직관적인 일정 투표 인터페이스 제공으로 서비스 활용도 증대.
